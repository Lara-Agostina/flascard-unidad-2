Unidad 2: Procesos de Desarrollo Ágil

1. Características del Proceso Ágil

  • Incremental e Iterativo
  • Colaborativo (Cliente + Equipo)
  • Respuesta rápida al cambio
  • Entrega continua de valor

2. Etapas Genéricas

Planificación (Sprint Planning) → Desarrollo → Pruebas → Revisión (Demo) → Retrospectiva.

3. Roles y Funciones

Product Owner: Define el "Qué" (Backlog).
Scrum Master: Facilita el proceso.
Equipo: Ejecuta el "Cómo".

4. Ceremonias Ágiles

Planning, Daily (15 min), Review (con clientes) y Retrospective (mejora interna).

5. Requisitos F vs NF

Funcionales: Comportamiento (ej. Registrar usuario).
No Funcionales: Atributos de calidad (ej. Seguridad, tiempo de respuesta).

6. Historias de Usuario (INVEST)

Independent, Negotiable, Valuable, Estimable, Small, Testable.

7. Priorización MoSCoW

Must, Should, Could, Won't have this time.
